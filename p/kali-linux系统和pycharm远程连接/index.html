Kali linux系统和pycharm远程连接

2025-03-03
阅读时长: 2 分钟

一、开启kali

在使用 PyCharm 连接 Kali 之前，需要保证 Kali 虚拟机的 SSH 服务处于开启状态。你可以按照以下步骤操作：

  1. 开启 SSH 服务：sudo systemctl start ssh或 sudo service ssh status
  2. 查看服务状态：输入 sudo systemctl status ssh 来查看 SSH 服务是否已成功开启。若显示 active (running)，就表明服务已正常运行。
  3. 查看 IP 地址：输入 ip addr 或者 ifconfig 命令查看 Kali 虚拟机的 IP 地址

二、实现Kali linux系统和pycharm远程连接 2021.3.1专业版pycharm

（1）在pycharm软件中创建连接记录 菜单栏Tools > Deployment > Configuration > + > SFTP > kali的ip地址 > OK （2）配置连接记录，如图1-16所示，表示连接成功。 ··· > + > 配置ip、用户名、密码 > 测试连接

（3）配置部署关联目录 /home/kali/pythonProject 若没有该目录，则在kali中新建 local path ： 本地目录 > Deployment path： /home/kali/文件夹

（4）配置项目解释器 导航栏File > setting > Project code > + Add Content > + > SSH Interpreter > 选择Existing serber configureation > ··· > 选择创建的root@地址 > Move

设置 Sync folder为 <Project root>-/home/kali/文件夹>OK>OK

提示：一定要确认映射文件目录是否有错，否则解释器失效。 （5）同步pycharm代码到KAli系统，即完成准备工作。

若报错没有nmap包，则在kali下载 pip install python-nmap

三、两种方法二选一即可。拿到root权限

Operation not permitted 表示没有权限，解决方法：

方法一、在Pycharm的terminal 选择远程的环境

  1. 切换root：sudo su 密码：kali
  2. 在代码路径下运行： 进入路径：cd kali中代码的路径 运行代码：python scapy01.py

方法二、允许root用户远程登录

  1. 修改配置文件/etc/ssh/sshd_config，允许root用户远程登录。 sudo vim /etc/ssh/sshd_config #允许root用户远程登录 #rmitRootLogin prohibit-password需修改为：（注意：#要删掉） PermitRootLogin yes

  2. 保存并退出后，重新启动ssh服务。 sudo systemctl restart ssh

  3. 给root用户创建密码 root用户是linux中的超级管理员，可以操作所有的资源。默认情况下，root用户是无法登陆的，需要我们自己来创建密码。故此在测试远程连接之前，需要给root用户创建密码。 代码为：sudo passwd root 密码统一设为root 账户之间进行用户切换采用su - 用户名。

后面远程连接时则可以使用root，root连接。
